Quero implementar três melhorias no projeto UniFarma, mantendo toda a arquitetura atual, o padrão de código existente e sem quebrar nenhuma funcionalidade.

Antes de alterar qualquer arquivo, analise a estrutura completa do projeto para identificar a melhor forma de implementar essas mudanças.

## 1. Sistema de Login (Prioridade Máxima)

Atualmente o sistema possui um login de administrador já configurado.

Quero adicionar um sistema de cadastro de usuários, porém o usuário administrador existente NÃO pode ser removido, alterado ou deixar de funcionar.

Requisitos:

* Manter o login Admin atual funcionando exatamente como hoje.
* Criar funcionalidade de cadastro de novos usuários.
* Permitir que novos usuários façam login normalmente.
* Armazenar as senhas de forma segura (hash).
* Utilizar a estrutura de autenticação já existente, caso exista.
* Não remover nenhuma permissão já implementada.
* Não quebrar o fluxo atual do sistema.

Caso seja necessário alterar banco de dados, criar automaticamente a migration correspondente.

---

## 2. Registrar qual usuário realizou cada atendimento

Quero que todo atendimento realizado no UniFarma fique vinculado ao usuário autenticado.

Requisitos:

* Ao criar um atendimento, registrar automaticamente o usuário logado.
* Salvar essa informação no banco de dados.
* Exibir na visualização do atendimento:

  * Nome do usuário que realizou o atendimento.
* Caso exista histórico ou tela de listagem, incluir essa informação quando fizer sentido.
* Não permitir que essa informação seja preenchida manualmente pelo usuário; ela deve ser obtida automaticamente a partir da autenticação.

---

## 3. Corrigir bug na impressão de atendimentos grandes

Existe um problema na renderização da impressão.

Quando o atendimento possui poucos medicamentos, a impressão funciona normalmente.

Porém, quando existem muitos itens, a visualização de impressão fica quebrada:

* alguns medicamentos desaparecem;
* informações do paciente ficam desalinhadas;
* partes do conteúdo são cortadas;
* a impressão não representa corretamente o atendimento.

Quero que a impressão funcione corretamente independentemente da quantidade de medicamentos.

Analise cuidadosamente o componente responsável pela impressão e corrija o layout para que:

* todos os medicamentos apareçam;
* nenhuma informação seja cortada;
* o cabeçalho do paciente permaneça corretamente alinhado;
* o conteúdo seja distribuído corretamente entre páginas quando necessário;
* a impressão funcione tanto para poucos itens quanto para dezenas de itens.

Evite soluções que apenas aumentem a altura do componente. O objetivo é corrigir a renderização de impressão utilizando boas práticas de CSS para mídia `print`, paginação e quebra de páginas.
