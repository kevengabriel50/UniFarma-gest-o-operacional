# Implementação de Etiquetas, Estoque e Melhorias no Cadastro de Medicamentos

Quero implementar uma nova etapa no módulo de medicamentos do UniFarma. O objetivo principal desta atualização é criar um sistema de impressão de etiquetas simplificadas para uso operacional junto ao Soul MV, além de melhorar o controle de estoque.

## Contexto

O UniFarma não possui integração com o Soul MV e não precisa possuir.

O objetivo é apenas armazenar e reutilizar códigos de barras já existentes no hospital.

Na prática, o usuário cadastrará os medicamentos utilizando os mesmos códigos de barras que já existem nas etiquetas geradas pelo Soul MV.

Posteriormente, o UniFarma deverá ser capaz de imprimir uma cópia simplificada dessas etiquetas para agilizar processos operacionais.

O foco desta atualização deve ser a funcionalidade de impressão de etiquetas.

---

# Melhorias no Cadastro de Medicamentos

Adicionar os seguintes campos:

### Obrigatórios

* Nome do medicamento
* Código de barras
* Código interno
* Quantidade em estoque

### Opcionais

* Descrição
* Laboratório/Fabricante

Os campos opcionais não devem impedir o cadastro.

---

# Controle de Estoque

Cada medicamento deve possuir um campo de quantidade em estoque.

Exemplo:

* Cetorolaco → 15 unidades
* Dexametasona → 8 unidades
* Dipirona → 32 unidades

A quantidade deverá ser armazenada no banco de dados de forma permanente.

Essas informações não podem ser perdidas após reinicializações, deploys ou atualizações do projeto.

---

# Avisos de Estoque Baixo

Adicionar ao Dashboard uma área de alertas de estoque.

Exibir automaticamente medicamentos com quantidade baixa.

Sugestão:

* Menos de 10 unidades → alerta amarelo
* Menos de 5 unidades → alerta vermelho

O objetivo é permitir visualização rápida dos itens que estão acabando.

---

# Nova Funcionalidade: Etiquetas

Criar uma funcionalidade de impressão de etiquetas dentro do módulo de medicamentos.

Cada medicamento deverá possuir um botão:

"Imprimir Etiqueta"

Ao clicar, o sistema deverá gerar uma etiqueta simples e limpa.

Não é necessário reproduzir o layout completo do Soul MV.

O objetivo é apenas gerar uma etiqueta funcional para leitura por scanner.

---

# Layout da Etiqueta

Formato simples.

Exemplo:

DIPIRONA 1G/2ML

[CÓDIGO DE BARRAS]

2000003772513

ou

CETOROLACO 30MG

[CÓDIGO DE BARRAS]

2000003777747

Requisitos:

* Nome do medicamento em destaque
* Código de barras centralizado
* Número do código abaixo do código de barras
* Layout limpo
* Sem excesso de informações
* Compatível com impressão em etiquetas pequenas

Não incluir:

* Anvisa
* Fornecedor
* Lote
* Validade
* Dados fiscais

Essas informações não são necessárias para o objetivo desta funcionalidade.

---

# Impressão

Implementar utilizando o sistema de impressão do navegador.

Fluxo esperado:

Medicamentos
→ Selecionar medicamento
→ Imprimir etiqueta
→ Abrir tela de impressão do Windows
→ Usuário seleciona a impressora Zebra

Utilizar:

window.print()

Não implementar integração direta com a impressora neste momento.

O sistema deve funcionar com qualquer impressora configurada no Windows.

---

# Compatibilidade com Zebra

O layout deve ser pensado principalmente para impressoras Zebra de etiquetas.

A aplicação deve gerar etiquetas com dimensões reduzidas e boa legibilidade.

O código de barras deve ser gerado a partir do número armazenado no cadastro do medicamento.

O scanner deve conseguir ler normalmente o código impresso.

---

# Persistência

Todos os dados devem ser armazenados em banco de dados.

Os seguintes registros nunca devem ser perdidos:

* Medicamentos cadastrados
* Códigos de barras
* Códigos internos
* Quantidades em estoque
* Histórico relacionado aos medicamentos

Evitar qualquer armazenamento apenas em memória de sessão.

Todas as informações precisam permanecer disponíveis mesmo após reinicializações do sistema.

---

# Objetivo Final

Permitir que o UniFarma funcione como um apoio operacional para impressão rápida de etiquetas e consulta de medicamentos, reduzindo processos manuais e agilizando atendimentos sem depender de integração direta com o Soul MV.
